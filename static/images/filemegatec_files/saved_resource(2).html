Word wide web: rete di risorse informative interconnesse. Queste risorse sono mantenute da calcolatori detti server e accessibili grazie a software detti client.
server software che accetta connessioni e fornisce un servizio tramite protocolli.
server hardware dedicato all’esecuzione di uno o piu software di tipo server.
L’accesso alle risorse avviene tramite software aventi funzionalità di client
Esempio di client: firefox, gmail. Paradigma di comunicazione sulla rete è client-server.
c-s: client origina la connessione collegandosi ad un server. Server aspetta, accetta connessione e fornisce servizio. Client e server usano lo stesso protocollo che stabilisce regole di comportamento delle 2 parti.
Protocolli usati sono in cima alla pila iso/osi e tcp/ip. Sono di tipo application. Per la navigazione web usiamo protocollo http: browser richiede risorsa a un server, server risponde con notifica di successo o errore seguita poi dalla richiesta.
contentware: sono contenuti ovvero i dati e le informazioni disponibili sulla rete tramite i server ed accessibili tramite i browser.
ipertesto: insieme di testi o pagine leggibili tramite interfaccia elettronica. È definibile come grafo con nodi: raggruppamento dati e archi collegamento tra dati. All’interno dell’ipertesto troviamo microtesti (nodi) legati tra loro tramite link.
multimediale: uso PC per presentare audio, testo, animazioni. Mediante PC. (es smartphone)
ipermedia: testo, audio, grafica sono interconnessi creando una raccolta di informazioni (es www).
documenti multimediali comprendono: testo, immagini, audio, video.
documento ipermediale è come web. Nodi possono essere costituiti da qualsiasi tipo di dato. Nodi sono individuati tramite URL: sequenza di caratteri stampabili. <schema>:<dettagli specifici>
Risorse raggiungibili tramite protocollo http sono identificate sottoforma di URL:
http://<nodo>:<porta>/<percorso>?<query>#<frammento>
nodo: indirizzo IP o nome server; porta: porta socket su cui server è in ascolto
percorso: percorso del file relativo al file system del server.
client ci permette interazione con server. Nel caso del web il client è il browser (interfaccia grafica, visualizzazione). Le risorse web ben gestite sono usufruibili da piu client e da qualsiasi tipo di browser.

Se il servizio web deve interagire con dati persistenti allora è necessario l’uso di un sistema di memorizzazione (storage). Lo scopo del database è di raccogliere dei dati e offrirli a utenti o altri componenti software.
Il modello logico di un database descrive come i dati sono rappresentati (relazionale, gerarchico …)
Database Managemennt System (DBMS) si occupa della gestione e dell’astrazione del database
CRUD: create, read, update, delete è una funzionalità del DBMS. L’altra sono le query
DATABASE RELAZIONALI: organizzati in tabelle dove ogni riga è un record e ogni colonna/tupla è attributo
DATABASE NON RELAZIONALI: organizzati in documento, grafi, coppie chiave/valore, o anche tabelle
sono usati per dati non strutturati o che non conviene strutturare. Si progettano tenendo conto delle query
i dati sono rappresentati da un documento JSON.
I non relazionali sono piu scalabili ed esosi rispetto ai SQL.
DATABASE NOSQL: non relazionali, semplici. Si usa una chiave per recuperare i valori. Rinunciano a proprietà ACID. Danno migliori performance in lettura e scrittura, query complesse, processano parallelamente i dati, tipi di dato flessibili
MONGODB è un DBMS non relazionale orientato ai documenti. È NOSQL. Nato tra 2007 e 2009
Phyton è il suo linguaggio di riferimento. Funziona come ORM senza necessità di fare mapping tra oggetti e modello relazionale. Accedo a dati tramite query, indici, collezioni. I documenti sono scritti in JSON
